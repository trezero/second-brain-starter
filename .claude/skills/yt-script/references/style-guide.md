## Title Patterns

### Formats That Work

1. **"The X of Y"** - Distilling complex topics
   - "The 80/20 of Building AI Agents"
   - "The Problem with MCP in 5 Minutes"

2. **Question hooks** - Create curiosity
   - "Are Agent Harnesses Bringing Back Vibe Coding?"

3. **Numbers/Lists** - Promise specific value
   - "Everything That Happened with AI in 2025"
   - "Every RAG Strategy Explained in Under 15 Minutes"

4. **"How to" / Guides** - Clear utility
   - "How to Build Your Own AI Coding Workflow"

5. **Technology focus** - Name the tool
   - "Docker just Made Using MCP Servers 100x Easier"
   - "Build Your First Voice AI Agent in 15 Minutes with LiveKit"

6. **Time-based** - Promise efficiency
   - "Build Your First Voice AI Agent in 15 Minutes"
   - "Get Started with the Claude Agent SDK in 10 Minutes"

7. **Bold claims with parenthetical clarification**
   - "GitHub is the Future of AI Coding (Here's Why)"
   - "2026 is Going to be Crazy for AI (Here's Why)"
   - "The True Power of AI Coding - Build Your OWN Workflows (Full Guide)"

### Title Rules

- NOT too clickbaity but very interesting
- Natural sounding, not robotic
- Include the main technology/topic
- Parentheticals add context without being cringe
- Can include CAPS for emphasis sparingly (FULL, EASIEST, ANY)

### Title Examples by Type

**Tutorial/Build videos:**
- "Build Your First RAG Agent with Pydantic AI + MongoDB + Docling"
- "Build a RAG Agent with Real-Time Source Validation"

**Explanation videos:**
- "The Problem with MCP in 5 Minutes"
- "All RAG Strategies Explained in 10 Minutes"

**Opinion/Prediction videos:**
- "2026 is Going to be a Crazy Year for AI - Here are My Predictions"
- "Hot Takes for AI Going into 2026"

**Tool showcase videos:**
- "Docker just Made Using MCP Servers 100x Easier (One Click Installs!)"
- "Turn ANY File into LLM Knowledge in SECONDS"

## Thumbnail Text Patterns

### Characteristics
- 2-5 words maximum
- Action-oriented or benefit-focused
- Can be a phrase, not necessarily grammatically complete
- One option is always **bolded** as the selection

### Examples by Category

**Action-oriented:**
- "Build a RAG Agent"
- "Build With Me"
- "Connect ANY Agent"

**Benefit-focused:**
- "RAG You Can Trust"
- "Custom Voice Agents"
- "AI Agents Made Simple"

**Topic labels:**
- "Claude Agent SDK"
- "MCP Command Center"
- "2026 AI Predictions"

**Intrigue/Hook:**
- "Agent Evolution"
- "The Sauce"
- "It's Inevitable..."

**Sometimes:**
- "No text" is a valid option

## Audience Reactions Patterns

### Purpose
Capture what the creator WANTS viewers to resonate with. These are aspirational comments that guide the content creation.

### Structure
- 3-5 reactions per script
- Written as actual YouTube comments would read
- Mix of gratitude, excitement, and value acknowledgment

### Patterns

**Gratitude + Specific Value:**
- "Dude this is exactly what I needed for my agent, thank you so much!"
- "Man I've been sick of using all of these expensive platforms like Vapi - thank you so much for this!"

**Quality Recognition:**
- "Best breakdown of AI agents I've ever seen on YouTube."
- "Such a clear guide - please do more like this!"

**Personal Impact:**
- "This makes so much sense, love the categories that you went through here to make it clear."
- "So many things clicked for me in this video"

**Future Action:**
- "Beautiful walkthrough, gonna go build my first agent now!"
- "100% going to be using this myself to make quick changes to my website on the fly!"

**Request for More:**
- "More videos on XYZ!"
- "Please do more content like this"

**Validation of Approach:**
- "Principles versus tools, I love it."
- "I 100% resonate with the mental models here"

## Description Patterns

### Structure
- 2-3 paragraphs (max 4 small ones)
- Strong hook in first sentence
- Problem/opportunity framing
- Value proposition
- What viewer will learn

### Opening Hooks (First Sentence Patterns)

**Problem identification:**
- "One of the biggest challenges we face with LLMs is their knowledge is too general and limited for anything new."
- "Finding and running MCP servers has always been a bit scattered."
- "Voice AI platforms like Vapi, Synthflow, and Bland are great for getting started quickly, but they come with trade-offs"

**Bold statement:**
- "The future of software is leaving most if not all of the coding to AI - I will die on that hill."
- "The IDE is dying. So is tool calling. OpenAI won't win."
- "Agent harnesses are making AI agents reliable - finally."

**Value promise:**
- "At this point I've covered basically all of the technologies I use in my tech stack throughout my content, but I've never done a single video sharing what makes up my entire tech stack and why."

### Middle Section
- Expand on the problem/opportunity
- Introduce the solution
- Name specific technologies/tools
- Build anticipation

### Closing (What They'll Learn)
- "In this video, I'll show you..."
- "I'll walk you through..."
- "Plus at the end, I'll even show you..."

### Example Full Description

```
One of the biggest problems with RAG systems is you never know what sources the AI Agent actually used. Did it hallucinate? Did it pull from irrelevant docs? And no, citation is not enough.

In this video, I show you how to solve that by building a RAG agent with human in the loop and source validation - users see the retrieved docs/chunks, approve or reject them, and only then does the agent synthesize an answer. This is especially powerful to have as a tool in your toolbelt as you are building RAG agents and refining them!

CopilotKit just dropped v1.50 with a complete AG-UI native rewrite, and it's perfect for this. I'm using their new useAgent hook for real-time state sync between a Pydantic AI backend and React frontend, plus human-in-the-loop that let users control exactly what sources get used.
```

## Outline Patterns

### Structure

```
- Intro - use description.
- [Main content sections with bullet points]
  - Sub-bullets for details
  - Notes like "show X", "demo Y"
- [Midroll placement if sponsored]
- [More content]
- Outro - [brief closing note]
```

### Common Elements

**Intro line:**
- "Intro - use description."
- "Intro - use description, focus on [specific aspect]"
- "Intro - use outline."

**Demo placements:**
- "Show demo of the agent"
- "Quick demo in the playground"
- "Final demonstration"

**Code walkthrough notes:**
- "Show the code at a high level"
- "Dive into React code"
- "Show workflow YML for Claude Code"

**Sponsor/Midroll:**
- "Midroll after slide 5"
- "Zapier Midroll"

**Outro patterns:**
- "Outro - just keep it simple!"
- "Outro - let me know in the comments what you want me to create next!"
- "Outro - so much more content I could cover with [topic]"

## Intro Patterns

### Key Differences from Description
- Written FOR SPEAKING, not reading
- More casual, conversational
- Can include emphasis with **bold**
- Same hook but worded naturally
- Can include `[Show X]` visual cues

### Transformation Example

**Description:**
> One of the biggest problems with RAG systems is you never know what sources the AI Agent actually used.

**Intro:**
> One of the biggest problems with RAG systems is you never know what sources the AI Agent actually used. Did it hallucinate? Did it pull from irrelevant docs? And no, citation is not enough.
>
> Today, I'll show you how to solve that by building a RAG agent with human in the loop...

### Intro-Specific Additions
- Rhetorical questions to the viewer
- "And yeah, call me crazy for..."
- "Alright so check this out."
- "And you heard that right..."
- Visual cues: `[Show quick demo]`, `[Click into page]`

### Common Intro Phrases
- "In this video, I'll walk you through..."
- "That's what I'll show you in this video!"
- "So with that, let's get right into it!"
- "If you've been wanting to [X] but found other approaches overcomplicated, I think you'll like this one."

## Overall Tone

### Voice Characteristics

**Direct and confident:**
- Make bold claims but back them up
- "I will die on that hill"
- "There is no denying that"

**Approachable and relatable:**
- "I know it's tempting to just wing it"
- "honestly I've regretted using them in the past"
- Self-aware: "I know sometimes I make it seem like the only AI coding assistant on the planet"

**Value-focused:**
- "I've got you covered"
- "I've got solid recommendations no matter the type"

### Signature Phrases
- "And here's the thing"
- "But honestly"
- "I've got you covered"
- "This my friend is..."
- "Let me show you"
- "That's what I'll show you in this video"
- "The people who are the most successful are..."
- "If you want more control..."

### What to AVOID
- Robotic/LLM-sounding language
- Overly clickbaity promises
- Cringe phrases or excessive hype
- Generic filler content
- Condescending explanations
- Excessive exclamation marks
- "In this comprehensive guide..." type openings
